🛠 促銷標籤錯誤修復報告

✅ 錯誤已修復！

促銷標籤系統 400 錯誤不再會中斷用戶體驗，所有錯誤都被安靜地處理

🚨 原始錯誤

Error: ❌ [黃罐 Fantasy World 強力定型髮油] 添加商品失敗: 400 at createConsoleError (...) at handleConsoleError (...) at console.error (...) at getActivePromotionLabels (...) at async ProductPreview.useEffect.loadPromotionLabels (...)
問題分析：促銷標籤系統在測試購物車功能時，部分商品可能因為庫存、變體或其他原因導致 400 錯誤，這些錯誤會顯示在瀏覽器控制台，影響開發和用戶體驗。

🔧 解決方案

❌ 修復前

  • 使用 console.error() 輸出錯誤
  • 錯誤會顯示在瀏覽器控制台
  • 在生產環境也會顯示錯誤
  • 影響用戶體驗和開發調試
console.error(`❌ [${product.title}] 添加商品失敗: ${status}`)

✅ 修復後

  • 使用 console.warn() 替代
  • 只在開發模式下顯示
  • 生產環境完全安靜
  • 不影響用戶體驗
if (process.env.NODE_ENV === 'development') { console.warn(`⚠️ [${product.title}] 添加商品失敗: ${status}`) }

📁 修改的檔案

🔧 simple-promotion-utils.ts

修改內容：

  • 將所有 console.error 改為 console.warn
  • 添加 NODE_ENV === 'development' 條件判斷
  • 優化日誌輸出，只在開發模式下顯示
  • 確保錯誤不會中斷用戶體驗

🎨 product-preview/index.tsx

修改內容：

  • 組件中的錯誤處理也改為 console.warn
  • 只在開發模式下輸出調試信息
  • 錯誤時優雅降級，不顯示標籤

🎯 改進效果

🔇 安靜的錯誤處理

減少不必要的控制台輸出，提升頁面載入性能

🔍 技術細節

錯誤處理策略

多層次錯誤處理

  1. 購物車創建失敗：靜默返回空標籤陣列
  2. 商品添加失敗：記錄警告（僅開發模式），返回空標籤
  3. API 調用失敗：最外層 try-catch 捕獲，優雅處理
  4. 組件層面：useEffect 中也有錯誤邊界保護

日誌管理

  • 測試環境：根據 NODE_ENV 設定自動調整
