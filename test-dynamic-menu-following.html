🧪 動態跟隨測試控制

跑馬燈狀態: 顯示 | 選單位置: 計算中...

🎯 測試重點

  • 動態跟隨：手機選單會實時跟隨導覽列位置
  • 跑馬燈切換：當跑馬燈隱藏/顯示時，選單位置自動調整
  • 無縫對齊：選單上邊界始終貼齊導覽列下邊界
  • 平滑過渡：位置變化有平滑的動畫效果

🔧 實現方式

  • 使用 getBoundingClientRect() 獲取導覽列實際位置
  • 用 MutationObserver 監聽 DOM 結構變化
  • 監聽視窗 resize 事件處理響應式變化
  • 動態計算跑馬燈高度並即時更新選單位置
  • 在選單開啟時重新計算確保位置準確

📱 測試步驟

  • 1. 點擊「開啟手機選單」觀察初始位置
  • 2. 點擊「切換跑馬燈顯示/隱藏」看選單是否跟隨
  • 3. 重複切換跑馬燈狀態測試動態跟隨
  • 4. 縮放瀏覽器視窗測試響應式適應

這是測試內容區域。請使用上方的控制按鈕來測試手機選單的動態跟隨效果。

選單位置會根據跑馬燈的顯示/隱藏狀態動態調整，始終保持與導覽列底部對齊。
