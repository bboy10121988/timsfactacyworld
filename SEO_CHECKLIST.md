```markdown
# SEO 實施檢查清單

## ✅ 技術要求 (Google Search Essentials)

### 基本要求
- [ ] **Googlebot 可存取**: 檢查 robots.txt 不會阻擋重要頁面
- [ ] **HTTP 200 狀態碼**: 所有重要頁面都返回成功狀態
- [ ] **可索引內容**: 頁面包含文字內容且未設定 noindex
- [ ] **HTTPS 安全連線**: 整站啟用 SSL 憑證
- [ ] **行動裝置友善**: 響應式設計且可在手機正常瀏覽

### Meta 標籤設定
- [ ] **頁面標題 `<title>`**: 每頁唯一，50-60 字元
- [ ] **Meta 描述**: 每頁唯一，140-160 字元，吸引點擊
- [ ] **Canonical URL**: 重複內容頁面設定標準網址
- [ ] **語言標籤**: `<html lang="zh-TW">` 設定正確語言
- [ ] **Viewport**: `<meta name="viewport">` 支援行動裝置

## 🎯 Core Web Vitals (核心網頁指標)

### 效能指標
- [ ] **LCP ≤ 2.5s**: 最大內容繪製時間
- [ ] **INP ≤ 200ms**: 互動至下次繪製時間  
- [ ] **CLS ≤ 0.1**: 累積版面配置位移

### 優化策略
- [ ] **預載入關鍵資源**: 使用 `<link rel="preload">`
- [ ] **圖片優化**: WebP/AVIF 格式，適當尺寸
- [ ] **字體優化**: 使用 `font-display: swap`
- [ ] **JavaScript 優化**: Code splitting, lazy loading
- [ ] **CSS 優化**: 移除未使用樣式，Critical CSS

## 📊 結構化資料 (Schema.org)

### 基本設定
- [ ] **JSON-LD 格式**: 使用推薦的結構化資料格式
- [ ] **適合的 Schema 類型**: Article, Product, Organization 等
- [ ] **完整資訊**: 包含必要屬性如 name, description, image
- [ ] **測試通過**: 使用 Google Rich Results Test 驗證

### 常用類型
- [ ] **文章頁面**: NewsArticle, BlogPosting
- [ ] **產品頁面**: Product, Offer
- [ ] **組織資訊**: Organization, LocalBusiness
- [ ] **麵包屑**: BreadcrumbList
- [ ] **常見問題**: FAQPage

## 🔗 內部連結與導航

### 連結結構
- [ ] **清晰的層級結構**: 首頁 → 分類 → 子分類 → 內容
- [ ] **有意義的錨文字**: 避免「點擊這裡」，使用描述性文字
- [ ] **內部連結**: 相關內容互相連結
- [ ] **麵包屑導航**: 幫助用戶理解位置
- [ ] **404 頁面**: 提供有用的錯誤頁面

### 技術實作
- [ ] **普通 `<a>` 標籤**: 不使用 JavaScript 模擬連結
- [ ] **相對路徑**: 內部連結使用相對路徑
- [ ] **nofollow 適當使用**: 外部或付費連結加上 nofollow

## 🖼️ 圖片 SEO

### 基本要求
- [ ] **Alt 屬性**: 所有圖片都有描述性替代文字
- [ ] **適當檔名**: 使用描述性檔名而非 IMG001.jpg
- [ ] **適當尺寸**: 根據顯示尺寸優化圖片大小
- [ ] **現代格式**: 使用 WebP 或 AVIF 格式
- [ ] **載入優化**: 關鍵圖片標記 `fetchpriority="high"`

### Social Media
- [ ] **Open Graph 圖片**: 1200x630 像素
- [ ] **Twitter Card 圖片**: 適當尺寸
- [ ] **Favicon**: 多種尺寸的 favicon

## 📱 社群媒體優化

### Open Graph
- [ ] **og:title**: 社群分享標題
- [ ] **og:description**: 社群分享描述  
- [ ] **og:image**: 社群分享圖片
- [ ] **og:url**: 標準網址
- [ ] **og:type**: 內容類型 (website, article 等)

### Twitter Cards
- [ ] **twitter:card**: 卡片類型
- [ ] **twitter:title**: Twitter 標題
- [ ] **twitter:description**: Twitter 描述
- [ ] **twitter:image**: Twitter 圖片

## 🤖 Robots 與 Sitemap

### Robots.txt
- [ ] **正確的 robots.txt**: 允許重要頁面，禁止不必要頁面
- [ ] **Sitemap 連結**: 在 robots.txt 中指定 sitemap
- [ ] **適當的 Crawl-delay**: 避免過度抓取

### XML Sitemap
- [ ] **完整的頁面列表**: 包含所有重要頁面
- [ ] **正確的優先級**: 重要頁面設定較高 priority
- [ ] **更新頻率**: 合理的 changefreq 設定
- [ ] **最後修改時間**: 準確的 lastmod 日期

## 🔍 關鍵字優化

### 研究與選擇
- [ ] **目標關鍵字**: 每頁設定主要關鍵字
- [ ] **長尾關鍵字**: 包含相關的長尾關鍵字
- [ ] **競爭分析**: 分析競爭對手的關鍵字策略
- [ ] **搜尋意圖**: 關鍵字符合用戶搜尋意圖

### 內容優化
- [ ] **標題包含關鍵字**: H1 包含主要關鍵字
- [ ] **內容中自然使用**: 避免關鍵字堆砌
- [ ] **相關關鍵字**: 使用語義相關的詞彙
- [ ] **URL 包含關鍵字**: 友善的 URL 結構

## 📈 監測與分析

### 工具設置
- [ ] **Google Search Console**: 監控搜尋表現
- [ ] **Google Analytics**: 分析流量與行為
- [ ] **Core Web Vitals 監控**: 追蹤效能指標
- [ ] **排名監控**: 追蹤關鍵字排名

### 定期檢查
- [ ] **索引狀態**: 檢查頁面是否被正確索引
- [ ] **抓取錯誤**: 修正 Googlebot 抓取問題
- [ ] **效能報告**: 監控 Core Web Vitals 趨勢
- [ ] **安全問題**: 檢查惡意軟體或駭客攻擊

## 🚫 避免的垃圾內容政策

### 禁止行為
- [ ] **避免關鍵字堆砌**: 不要過度重複關鍵字
- [ ] **避免隱藏文字**: 不使用隱藏文字或連結
- [ ] **避免門頁**: 不建立專為搜尋引擎的頁面
- [ ] **避免重複內容**: 每頁都有獨特內容
- [ ] **避免欺騙性重定向**: 不誤導用戶或搜尋引擎

### 內容品質
- [ ] **原創內容**: 創建有價值的原創內容
- [ ] **E-A-T**: 展現專業性、權威性、可信度
- [ ] **用戶體驗**: 優先考慮用戶需求而非搜尋引擎
- [ ] **定期更新**: 保持內容新鮮與相關
- [ ] **回應用戶**: 解決用戶問題與需求

---

## 🛠️ 實施指南

1. **優先處理技術要求**: 確保網站可被正常抓取和索引
2. **優化 Core Web Vitals**: 提升頁面載入速度和用戶體驗  
3. **完善內容 SEO**: 優化標題、描述、內容結構
4. **設置監控工具**: 建立追蹤和分析機制
5. **持續優化**: 根據數據持續改進 SEO 策略

使用此檢查清單確保您的網站完全符合 Google Search Essentials 並提供優秀的用戶體驗！
```
