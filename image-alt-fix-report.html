🖼️ Image Alt 屬性修正

解決 Next.js Image 組件缺少 alt 屬性的錯誤

❌ 錯誤分析

Error: Image is missing required "alt" property. Please add Alternative Text to describe the image for screen readers and search engines. at createConsoleError (http://localhost:8000/_next/static/chunks/node_modules_next_dist_client_8f19e6fb._.js:882:71) at Hero (http://localhost:8000/_next/static/chunks/src_f5dec9c0._.js:63:34)

錯誤來源

錯誤發生在 Hero 組件 中的 Image 元素，該組件位於首頁輪播圖部分。

🔍 問題定位

原始程式碼

<Image src={slideItem.backgroundImage} alt={slideItem.heading} fill className="object-cover object-center" sizes="(max-width: 640px) 100vw, (max-width: 768px) 100vw, 100vw" priority={index === 0} />

問題：當 slideItem.heading 為空或未定義時，alt 屬性會是空值。

修正後程式碼

<Image src={slideItem.backgroundImage} alt={slideItem.backgroundImageAlt || slideItem.heading || `輪播圖片 ${index + 1}`} fill className="object-cover object-center" sizes="(max-width: 640px) 100vw, (max-width: 768px) 100vw, 100vw" priority={index === 0} />

解決：使用邏輯運算子提供多重後備值，確保總有有效的 alt 文字。

🔧 修正策略

Alt 屬性優先順序

  1. 專用 Alt 文字： slideItem.backgroundImageAlt - 專為圖片設計的描述文字
  2. 標題文字： slideItem.heading - 輪播圖的標題作為描述
  3. 預設描述： `輪播圖片 ${index + 1}` - 最後的保險後備值

無障礙性考量

  • 螢幕閱讀器支援：確保視障用戶能理解圖片內容
  • SEO 優化：搜尋引擎能更好地理解圖片內容
  • 網頁標準遵循：符合 WCAG 無障礙指引
  • 開發體驗：避免 Next.js 在開發環境中的警告錯誤
