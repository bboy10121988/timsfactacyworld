🚚 宅配流程修復驗證報告

修正宅配不需要串接綠界物流選擇的問題

🚨 問題描述

原始問題：在結帳頁面中，選擇「宅配」後會錯誤地跳轉到綠界物流選擇頁面

預期行為：宅配應該直接進入第二步驟（填寫收件資料），不需要串接綠界

實際行為：宅配和超商取貨都會觸發 ECPay 物流選擇頁面

🔧 問題根因分析

在 checkout-form/index.tsx 的第 134-136 行，判斷是否需要 ECPay 物流選擇的邏輯錯誤：

❌ 修復前（錯誤邏輯）： const isEcpayLogistics = selectedMethod?.name?.includes('超商') || selectedMethod?.name?.includes('宅配') || // ← 問題在這裡 selectedShippingType === 'convenience_store' || selectedShippingType === 'home_delivery' // ← 和這裡
✅ 修復後（正確邏輯）：
const isEcpayLogistics = selectedMethod?.name?.includes('超商') || selectedShippingType === 'convenience_store' // 只有超商取貨需要串接綠界物流選擇，宅配直接進入下一步

❌ 修復前流程

1. 選擇配送方式：宅配
2. 🚨 錯誤跳轉到綠界物流選擇
3. 用戶困惑，流程中斷

✅ 修復後流程

1. 選擇配送方式：宅配
2. ✅ 直接進入收件資料填寫
3. 流程順暢繼續

🧪 測試驗證步驟

  1. 啟動前端服務：cd frontend && npm run dev
  2. 進入結帳頁面：加入商品到購物車，點擊結帳
  3. 測試宅配流程：
    • 選擇「宅配」配送方式
    • 點擊「確認選擇」按鈕
    • ✅ 應該直接進入第二步驟（收件資料填寫）
    • ❌ 不應該跳轉到綠界物流選擇頁面
  4. 測試超商取貨流程：
    • 選擇「超商取貨」配送方式
    • 點擊「確認選擇」按鈕
    • ✅ 應該跳轉到綠界物流選擇頁面
